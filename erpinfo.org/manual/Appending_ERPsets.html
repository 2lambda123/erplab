To add the bins from one or more ERPsets onto the end of another ERPset, you can select ERPLAB > Append ERPsets.Â  As shown in the screenshot below, you will specific the indexes of the ERPsets to be appended. When you do this, the bins from the first ERPset retain their original numbers (e.g., bin 1, bin 2, etc.), and the bins from the additional ERPsets are added onto the end. For example, if the first ERPset has 3 bins and the second ERPset has 2 bins, bins 1-3 in the new ERPset will be bins 1-3 from the first ERPset, and bins 4-5 in the new ERPset will be bins 1-2 from the second ERPset. The newly created ERPset will then become the active ERPset.

In many cases, the bins in the ERPsets being appended have identical names (e.g., the name of bin 1 in the first ERPset is the same as the name of bin 2 in the second ERPset). The Append ERPsets routine has a "prefix" option that can disambiguate these bins in the newly created ERPset. For example, if you create prefixes of File1 and File2, as shown in the screenshot, the bin labels from the first file will have a prefix of File1: and the bin labels from the second file will have a prefix of File2: in the newly created ERPset.
